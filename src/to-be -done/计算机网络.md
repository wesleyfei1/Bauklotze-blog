# Chap 4: Network layer

选路以及转发，计算转发表完成选路，然后由此进行转发

数据面: 在路由器内部实施分组转发

控制面: 选路是控制面功能(如何去计算这个路由表)，而为上层提供的服务就是网络服务模型



网络层服务:主机到主机，数据报网络(无连接)，以及虚电路网络(链接，也就是主机到主机的路径，记录所有的端到端路径，虚电路号以及相应的转发表项)，incoming&&outgoing interface&&VC



两种网络，支持signaling 协议(建立维护以及拆除)，以及数据包网络(只是目的地址于输出端口的映射)，数据包网络只提供最小服务(**可运行在各种链路上，设计灵活** )

**路由器**

- 输入端口:接受，data link layer,  查表(转发表)+排队+转发
- 交换设施(memory走总线+memory,bus(总线)，crossbar)
- 输出: 组装结构(组装成分组)，排队，调度+datalink/physical layer

- 分组丢弃原则: 尾部/随机丢弃RED(**limit1开始以p概率丢弃，limit2开始完全丢弃，p dynamic**)/优先级()几个stack,轮询调度+加权)，以及主动队列管理

**IP**

基本数据报的格式(IHL,Total length,Identi,DF&&MF,片偏移量，TTL),4字节，1字节，8字节的单位

以及如何组装报文



接口(路由器/主机和物理链路的边界，接口对应一个IP地址)
